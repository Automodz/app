## PRIORITY GAP LIST (what to build next)

### 🔴 HIGH PRIORITY (core functionality gaps)
1. **Booking slot conflict check** — Query Firestore before confirming to prevent double-booking
2. **Ceramic 2-day blocking** — When ceramic is selected, block the next day too
3. **Subscription wash deduction** — When a subscription customer books a wash, decrement `washesUsed`
4. **In-app notifications auto-created** — Write to `notifications` collection on booking create + status update
5. **Admin: Verify payment button** — One-click to set `paymentStatus: 'verified'` on a booking
6. **Customer gallery page** — `/dashboard/gallery` showing before/after from the gallery collection

### 🟡 MEDIUM PRIORITY (PRD-specified features)
7. **Referral discount at checkout** — Read referral record, show 15% off, apply to booking amount
8. **Admin: attach photos to booking** — Let admin paste before/after URLs on completed bookings
9. **Admin: per-customer drill-down** — Click customer → see their vehicles + full booking history
10. **Real-time status listener** — Replace `getDocs` with `onSnapshot` in history page
11. **Invoice view** — Simple printable booking summary page at `/dashboard/booking/[id]`
12. **Service history: show before/after** — Display booking-level photos when admin has uploaded them

### 🟢 LOWER PRIORITY (nice-to-have / QR / email)
13. **QR route** — `/qr` redirects to `/auth/register?ref=qr` for shop QR code
14. **OTP phone login** — Firebase Phone Auth (requires Blaze plan for SMS in India, or use WhatsApp OTP workaround)
15. **Email confirmation** — Use EmailJS (free) or Resend (free tier) — no backend needed
16. **Admin calendar view** — Week/day view for bookings
17. **Admin: block time slots** — Mark specific dates/times as unavailable
18. **Subscription auto-expiry check** — On login, check if `endDate` has passed and set status to `expired`

---

## WHAT'S SOLID ✅
The foundation is production-quality. Auth, vehicles, booking wizard, admin status management, subscriptions UI, referral UI, dark/light theme, PWA, and the premium design system are all done well. The app is **deployable today** — customers can register, add vehicles, book services, and track status. Admins can manage bookings and update status with WhatsApp notifications.

## WHAT'S MISSING AT A BUSINESS LEVEL ⚠️
The three things that would cause real operational problems are:
- No double-booking prevention
- Subscription washes not decrementing (members could use unlimited washes)
- Notifications collection never written to (notification centre always empty)
